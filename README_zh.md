# Scientific Figure Generator

一个**两层 Prompt 编译器**，将科学概念的自然语言描述转化为可发表的优质图表——具备引导式定制、多模型并行生成，以及可节省高达 60% token 的提示词蒸馏功能。

> **一句话总结：** 不要盲目地在网页 UI 里输入提示词然后祈祷。本 Skill 提供交互式、推荐驱动的工作流，理解你的学科领域，选择合适的布局，蒸馏提示词以获得最佳模型表现，并从 6 个模型并行生成图表，让你比较后选出最佳结果。

---

## 为什么需要这个工具

研究人员花费数小时与 AI 图像生成器搏斗以制作科学图表。典型的工作流是这样的：

| 痛点 | 实际情况 | 我们的解决方案 |
|---|---|---|
| **盲目提示** | 你在网页 UI 中输入描述，点击生成，然后祈祷。没有迭代、没有引导、没有结构。 | **5 个交互阶段**分析你的输入，推荐学科/风格/布局，让你确认或覆盖——专家引导而不剥夺你的控制权。 |
| **Token 浪费** | 你精心编写的 800 token 提示词在模型的注意力机制中被稀释。埋在第 3 段的关键约束实际上被忽略了。 | **两阶段提示词蒸馏**将提示词压缩 40–60%，把冗长的学术规则转化为图像模型真正能响应的具体视觉词汇。 |
| **单模型赌博** | 你选择一个模型（Midjourney？DALL·E？Gemini？），生成 4 个变体，然后凑合。无法知道另一个模型是否会做得更好。 | **6 个模型并行生成**——同样的蒸馏提示词，所有后端同时运行。输出文件名遵循 `[ID]_[round]_[provider].png` 格式，方便系统化比较。 |
| **布局猜测** | 你手动描述空间排列（"把 X 放在左边，Y 放在右边，Z 放在中间……"），但无法保证模型能理解。 | **自动布局引擎**支持 6 种布局类型（Pipeline、Central Hub、Grid、Flow、Radial、Hierarchical），自动分配视觉权重并优化阅读路径。 |
| **抽象规则失效** | 你写"确保一致性和可读性"——模型不知道那看起来是什么样。 | **视觉原语翻译**——"一致的实体颜色、灰度可读的调色板、≥30% 亮度差"，替代抽象指令。 |
| **领域无知** | 通用图像模型不懂生物信号通路惯例、ML 架构图标准或化学符号表示。 | **8 个学科库**提供领域特定的视觉语法：蛋白质形状、箭头语义、CPK 颜色惯例、层表示等。 |
| **没有质量基线** | 你必须记住在每个提示词中包含所有质量要求。 | **内置质量标准**——三秒法则、三层视觉层次、色盲友好调色板、元素数量纪律，每次生成都自动包含。 |

## 架构

```
                    ┌─────────────────────────────────┐
                    │  层 1：决策引擎                    │
                    │  （阶段 0–7，与模型无关）           │
                    │                                  │
用户输入 ───────────▶│ 阶段 0：接收                      │
                    │ 阶段 1：自动分析                   │
                    │ 阶段 2：学科选择                   │──▶ 输出：
                    │ 阶段 3：叙事模式                   │      • 蒸馏提示词
                    │ 阶段 4：风格选择                   │      • 布局类型
                    │ 阶段 5：自动布局                   │      • 风格
                    │ 阶段 6：提示词构建                 │      • 宽高比
                    │ 阶段 7：提示词蒸馏                 │
                    └──────────────┬──────────────────┘
                                   │
                                   ▼
                    ┌─────────────────────────────────┐
                    │  层 2：图像提供商                  │
                    │  （阶段 8，多模型并行）             │
                    │                                  │
    蒸馏提示词 ─────▶│  用户选择 N 个提供商               │
                    │  ┌─ ERNIE-Image-Turbo (百度)      │
                    │  ├─ wan2.7-image-pro (百炼)       │──▶ 输出：
                    │  ├─ qwen-image-2.0 (百炼)         │      [ID]_[round]_[prov].png
                    │  ├─ z-image-turbo (百炼)          │      用于跨模型比较
                    │  ├─ gemini-3-pro-image (API MART) │
                    │  └─ GPT-Image-2 (API MART)        │
                    └─────────────────────────────────┘
```

**层 1** 与模型无关——它将用户意图编译为优化后的提示词，不耦合到任何特定提供商。**层 2** 接收蒸馏后的提示词，并行发送到用户选择的模型，从而科学评估哪个模型能最好地渲染该概念。

### 8 阶段工作流

| 阶段 | 名称 | 类型 | 内容 |
|---|---|---|---|
| 0 | 接收 | 👤 用户 | 描述你的图表——自然语言、代码或结构化大纲 |
| 1 | 自动分析 | ⚙️ 自动 | 检测学科、叙事模式、布局类型和风格匹配度 |
| 2 | 引导式学科选择 | 👤 用户 | 确认或覆盖推荐的学科领域 |
| 3 | 叙事模式 | 👤 用户 | 选择机制图或图文摘要模式 |
| 4 | 引导式风格 | 👤 用户 | 从学科感知的推荐中选择视觉风格 |
| 5 | 自动布局 | ⚙️ 自动 | 确定最优布局类型、视觉权重和阅读路径 |
| 6 | 提示词构建 | ⚙️ 自动 | 组装包含所有约束的 7 层完整提示词 |
| 7 | 提示词蒸馏 | ⚙️ 自动 | 压缩 40–60%——将抽象规则转化为视觉词汇 |
| 8 | 多提供商生成 | 👤 用户 | 选择提供商，并行生成，比较结果 |

## 功能特性

### 学科库（8 个领域）

每个学科都附带领域特定的视觉惯例、标准元素词汇和颜色标准：

- **生物医学与分子生命科学** — 信号级联、蛋白质复合物、细胞区室
- **化学、材料与纳米科学** — 合成路线、晶体结构、能级图
- **AI 与计算机科学** — 神经网络架构、数据管线、注意力机制
- **工程与应用物理** — 器件示意图、电路图、MEMS 结构
- **临床医学与医疗保健** — 疾病机制、治疗流程、试验设计
- **环境、地球与气候科学** — 生物地球化学循环、生态系统模型、气候反馈
- **物理与数学** — 量子过程、时空图、流形可视化
- **综合科学** — 跨学科或新兴领域的回退选项

### 视觉风格（5 种模式）

- **经典矢量** — 简洁、扁平、期刊就绪（Nature/Cell/Science 级别）
- **手绘草图** — 白板/研究笔记本美学
- **极简信息图** — 高冲击力编辑传播
- **照片级 3D 渲染** — 用于封面艺术和新闻稿
- **复古科学插图** — 经典教科书蚀刻风格

### 叙事模式

- **机制图** — 解释某事物如何运作（因果链、过程流）
- **图文摘要** — 用单一引人注目的构图传达关键发现
- 外加 5 个子模式：架构、工作流、比较、分类和时间线

### 自动布局引擎（6 种类型）

| 布局类型 | 最适用于 | 视觉模式 |
|---|---|---|
| Pipeline | 工作流、级联、顺序过程 | 从左到右的线性流 |
| Central Hub | 框架论文、多模态系统 | 核心被模块环绕 |
| Grid | 比较、多条件实验 | 矩阵排列 |
| Flow | 循环、反馈回路、状态机 | 有向图 |
| Radial | 层级结构、系统发育树 | 同心或分支 |
| Hierarchical | 分类体系、组织结构 | 自上而下的树 |

### 提示词蒸馏

蒸馏引擎在将完整组装的提示词发送到图像模型之前进行压缩。关键转换：

- **元指令移除** — 删除"你必须确保……"和"重要的是……"
- **抽象 → 视觉词汇** — "保持颜色语义" → `一致的实体颜色，≥30% 亮度差`
- **多约束聚类** — 将相关规则合并为紧凑的视觉描述符
- **位置优化** — 关键约束放在前面，模型注意力最强的位置
- **结构化 → 空间化** — "网格布局，等间距" → `3×2 网格，24pt 等间距，左对齐`

结果：相同语义内容，token 减少 40–60%，模型遵从度显著提升。

### 质量标准（每次生成都内置）

- **三秒法则** — 一眼即可识别核心机制
- **三层视觉层次** — 主要 → 次要 → 三级注意力序列
- **色盲友好** — 兼容绿色弱视和红色弱视的调色板
- **元素数量纪律** — 3–7 个结构元素，必要时分组
- **单一流向** — 始终从左到右或从上到下

## 快速开始

### 1. 安装

将此目录复制到你的 Claude Code skills 文件夹：

```
~/.claude/skills/scientific-figure/
```

### 2. 配置 API 密钥

编辑 `config.local.json`（已 gitignore）：

```json
{
  "baidu_ai_studio": {
    "api_key": "<你的百度 API 密钥>",
    "base_url": "https://aistudio.baidu.com/llm/lmapi/v3/images/generations"
  },
  "alibaba_bailian": {
    "api_key": "<你的百炼 API 密钥>",
    "base_url": "https://dashscope.aliyuncs.com/api/v1/services/aigc/multimodal-generation/generation",
    "task_url": "https://dashscope.aliyuncs.com/api/v1/tasks"
  },
  "apimart": {
    "api_key": "<你的 API MART 密钥>",
    "base_url": "https://api.apimart.ai/v1/images/generations"
  }
}
```

你只需配置打算使用的提供商。

### 3. 生成

在 Claude Code 中调用本 Skill：

```
/scientific-figure
绘制 Transformer 注意力层的机制图，包含
多头注意力、层归一化和残差连接。
```

Skill 会分析你的输入，推荐学科/风格/布局，确认后从你选择的提供商并行生成图像。

### 4. 比较与迭代

输出文件遵循 `[promptID]_[round]_[provider].png` 命名。跨模型比较结果，在新一轮中优化描述，最终得到最佳图表。

### 独立脚本使用

你也可以直接调用 `generate_images.py`（零依赖，仅需 Python 标准库）：

```bash
python generate_images.py \
  --prompt-file .claude/distilled_prompt.txt \
  --providers ernie,qwen,gemini \
  --output-prefix 001_r1 \
  --aspect-ratio 1:1
```
