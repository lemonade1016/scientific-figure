## 1. 概述

**Scientific Figure Generator v2** 是一个 AI 驱动的科研配图自动生成系统，以 Claude Code Skill 的形式运行。它接收用户对科学机制、模型架构或研究概念的自然语言描述，经过 8 个阶段的智能分析与编译，最终调用多个商业图像生成模型并行出图，实现"同一 Prompt，多模型对比"的科学可视化工作流。

### 核心能力

- **智能理解**：自动检测学科领域、叙事模式、布局类型，无需用户手动选择所有参数
- **专家级 Prompt 编译**：7 层 Prompt 架构，涵盖角色设定、学科规范、风格美学、空间布局、叙事结构、质量标准和用户内容
- **Prompt 精馏**：将 800-1500 词的全量 Prompt 压缩 40-60%，在保留全部关键约束的前提下，适配图像生成模型的注意力机制
- **6 模型并行生成**：同一精馏 Prompt 同时发送至 ERNIE-Image-Turbo、wan2.7-image-pro、qwen-image-2.0、z-image-turbo、Gemini-3-Pro-Image、GPT-Image-2
- **跨模型对比**：科学评估不同模型在空间准确性、风格保真度、科学准确性、文字质量、整体冲击力五个维度的表现

### 适用场景

- 论文配图（机制图、模型架构图、流程示意图）
- Graphical Abstract / TOC 图像
- 会议海报主图
- 期刊封面提案
- 基金申请书总结图
- 学术社交媒体传播图

## 2. 核心架构：双层 Prompt 编译器

整个 Skill 分为两个独立层：

```
                    ┌─────────────────────────────────┐
                    │  LAYER 1: 决策引擎 (Decision Engine) │
                    │  Phase 0-7, 模型无关                │
                    │                                  │
用户输入 ──────────▶│ Phase 0: 需求采集                  │
                    │ Phase 1: 自动分析                  │
                    │ Phase 2: 学科选择                  │──▶ 输出:
                    │ Phase 3: 叙事模式                  │      · 精馏 Prompt
                    │ Phase 4: 风格选择                  │      · 布局类型
                    │ Phase 5: 自动布局                  │      · 风格
                    │ Phase 6: Prompt 构建               │      · 宽高比
                    │ Phase 7: Prompt 精馏               │
                    └──────────────┬──────────────────┘
                                   │
                                   ▼
                    ┌─────────────────────────────────┐
                    │  LAYER 2: 图像生成 (Image Providers)│
                    │  Phase 8, 多模型并行               │
                    │                                  │
    精馏 Prompt ───▶│  用户选择 N 个模型                  │
                    │  ┌─ ERNIE-Image-Turbo (百度)       │
                    │  ├─ wan2.7-image-pro (阿里百炼)    │──▶ 输出:
                    │  ├─ qwen-image-2.0 (阿里百炼)      │      [ID]_[round]_[provider].png
                    │  ├─ z-image-turbo (阿里百炼)       │      用于跨模型对比
                    │  ├─ gemini-3-pro-image (API MART)  │
                    │  └─ GPT-Image-2 (API MART)         │
                    └─────────────────────────────────┘
```

### Layer 1：决策引擎（Phase 0–7）

**模型无关**——只负责将用户意图编译为最优的图像生成 Prompt，不与任何特定模型耦合。包含智能检测、专家推荐、自动布局、Prompt 构建与精馏。

### Layer 2：图像生成（Phase 8）

**多模型并行**——将 Layer 1 产出的同一精馏 Prompt 发送至用户选择的多个商业模型，实现科学可控的跨模型对比。

### v2 相比 v1 的改进

| v1 问题 | v2 解决方案 | 对应模块 |
|---------|------------|---------|
| Prompt 过长，模型注意力稀释 | 两阶段：全量 Prompt → 精馏 Prompt | `prompt-distillation.md` |
| 用户盲选（无引导） | 自动分析 + 专家推荐 | Phase 0–4 |
| 无叙事结构 | 机制图 vs Graphical Abstract 双引擎 | `narrative-composer.md` |
| 无空间智能 | 6 种布局类型 + 视觉权重系统 | `layout-engine.md` |
| 单模型输出，无法对比 | Layer 2 多模型并行生成 | `api-integration.md` |

## 3. 八阶段工作流详解

```
USER INPUT
    │
    ▼
Phase 0: 需求采集 ─── 收集原始描述/代码
    │
    ▼
Phase 1: 自动分析 ─── 检测学科、叙事模式、布局类型、风格匹配
    │
    ▼
Phase 2: 学科选择 ─── 推荐 + 备选 → 用户确认
    │
    ▼
Phase 3: 叙事模式 ─── 机制图 vs Graphical Abstract → 用户确认
    │
    ▼
Phase 4: 风格选择 ─── 学科感知推荐 + 备选 → 用户确认
    │
    ▼
Phase 5: 自动布局 ─── 布局类型、视觉权重、阅读路径、间距
    │
    ▼
Phase 6: Prompt 构建 ─── 7 层全量 Prompt 组装
    │
    ▼
Phase 7: Prompt 精馏 ─── 压缩 40-60%，优化模型注意力
    │
    ▼
Phase 8: 多模型生成 ─── 用户选择 N 个模型，并行生成，交叉对比
```

**用户交互阶段**：Phase 0、2、3、4、8（共 5 次交互）
**系统自动阶段**：Phase 1、5、6、7（全程自动）

**Prompt 缓存目录**：Phase 6 和 Phase 7 产出的全量 Prompt 与精馏 Prompt 静默写入 `.claude/prompt_cache/` 目录，而非在对话中直接展示（避免占用 token）。用户可随时打开该目录下的文件查看或复制 Prompt 到其他工具使用。

### Phase 0：需求采集（用户交互）

**目标**：在提出任何选择问题之前，先完整收集用户的原始需求。

系统会向用户提问：

> "请描述你的需求，尽可能详细：
> - 你想可视化的机制、架构或概念是什么？
> - 有哪些关键组件及其关系？
> - 核心信息或创新点是什么？
> - 配图用途？（期刊投稿、预印本、Graphical Abstract、会议展示、封面等）
> - 如有目标期刊或会议，请说明。"

用户可以提供：
- 自然语言机制描述
- 模型架构代码（Python/PyTorch/TensorFlow）
- 结构化步骤大纲
- 目标期刊/会议信息
- 参考风格描述

**代码自动翻译**：如果用户提供代码，系统会自动解析模块名、数据流、特殊连接和创新组件，转换为自然语言描述。

---

### Phase 1：自动分析（系统自动）

系统对 Phase 0 收集的输入进行四维分析：

**1A. 学科检测**：扫描输入中的领域关键词，统计匹配数，选取最高匹配度的学科。

**1B. 叙事模式检测**：
- Graphical Abstract 触发词：`graphical abstract`、`TOC`、`cover`、`summary`、`highlight`
- 机制图触发词：`mechanism`、`pathway`、`process`、`pipeline`、`architecture`

**1C. 布局类型预检测**：扫描 6 种布局类型关键词，预计算最佳匹配。

**1D. 期刊/会议提取**：识别目标期刊名（Nature、Cell、Science、NeurIPS、ICML 等）或类型（预印本、会议、期刊、封面等）。

系统将输出分析摘要：

> **自动分析结果：**
> - **检测学科：** 生物医学与分子生命科学（置信度：HIGH）
> - **检测叙事模式：** 机制图
> - **检测布局类型：** 管线布局
> - **目标期刊：** Nature Biotechnology

---

### Phase 2：学科选择（用户交互）

系统基于 Phase 1 分析，**推荐**（而非罗列）学科领域，使用 `AskUserQuestion` 交互。

**推荐逻辑**：
- 若明确目标期刊且与某学科有已知映射 → 提升该学科权重
- 若检测置信度 HIGH → 直接推荐
- 若置信度 MEDIUM 或 LOW → 推荐前 2 名，由用户选择
- 始终保留用户覆盖选项

推荐选项标注 `⭐ 推荐（原因）`。

### Phase 3：叙事模式选择（用户交互）

系统展示检测到的叙事模式供确认。

**选项 1：机制图（Mechanism Diagram）**
- 因果链：输入 → 过程 → 交互 → 输出
- 解释"如何工作"
- 适用于：通路图、架构图、算法流程图、过程示意图

**选项 2：Graphical Abstract**
- 影响力弧：问题 → 创新 → 结果 → 意义
- 传达"做了什么以及为什么重要"
- 适用于：期刊 TOC 图、新闻稿配图、封面、社交媒体摘要

用户也可以选择 **组合模式**（Hybrid），上部 70% 为详细机制，下部 30% 为 Graphical Abstract 总结条。

### Phase 4：风格选择（用户交互）

系统基于"学科 + 叙事模式 + 目标期刊"三维矩阵，计算最优风格推荐。

**风格决策矩阵**：

| 场景 | 推荐风格 | 原因 |
|------|---------|------|
| 传统期刊 + 任意学科 | Classic Vector 或 Hybrid | 编辑接受度优先 |
| 预印本 + 任意学科 | Hand-Drawn Sketch 或 Minimal Infographic | 社交媒体传播力 |
| Graphical Abstract + 任意 | Minimal Infographic 或 Hybrid | 缩略图可读性 |
| 机制图 + AI/CS | Hand-Drawn Sketch | 领域趋势 |
| 机制图 + 生物医学 | Scientific Illustration 或 Classic Vector | 生物学准确性要求 |
| 机制图 + 化学/材料 | Classic Vector 或 3D Render | 分子可视化需求 |
| 会议海报 | Futuristic Tech 或 Minimal Infographic | 远距离视觉冲击 |
| 期刊封面 | 3D Render 或 Scientific Illustration | 最大视觉冲击 |
| 跨学科 | Hybrid | 多层信息层次 |

系统以 `AskUserQuestion` 呈现 7 种风格，推荐风格标注 `⭐ 推荐`，备选标注 `🔄 备选`。

### Phase 5：自动布局分析（系统自动）

系统自动执行 4 个子步骤，无需用户干预。

**5A. 配图类型分类**：将配图归入 6 种布局类型之一（详见第 7 节）。

**5B. 视觉权重计算**：提取所有命名组件，基于以下信号评分（1-10）：

| 信号 | 评分 |
|------|------|
| 标注为"novel"、"our"、"proposed"、"创新" | 10 |
| 标注为"first"、"breakthrough"、"首次" | 10 |
| 主要研究对象（目标蛋白、模型名、候选药物） | 9 |
| 标注为"critical"、"essential"、"关键" | 8 |
| 标注为"important"、"重要" | 7 |
| 系统标准组件 | 5 |
| 辅助元素 | 3 |
| 环境上下文 | 3 |
| 可选附加元素 | 2 |

**5C. 阅读路径分配**：基于布局类型确定视觉流。

**5D. 布局 Prompt 组装**：生成布局指令，作为 Phase 6 的 Layer 4。

系统简要输出布局分析结果。

### Phase 6：全量 Prompt 构建（系统自动）

系统按 **7 层架构** 组装 Prompt：

```
LAYER 1: 角色与身份设定    —— 20 年经验科学插画师角色
LAYER 2: 学科特定规范      —— 从 discipline-library 加载
LAYER 3: 风格美学指令      —— 从 style-library 加载
LAYER 4: 自动布局空间指令   —— Phase 5 产出
LAYER 5: 叙事结构          —— 从 narrative-composer 加载
LAYER 6: 顶级质量标准      —— 从 quality-standards 加载
LAYER 7: 用户具体内容      —— 组件、关系、核心信息
```

全量 Prompt（约 800-1500 词）因篇幅较长，默认**静默写入缓存文件**而非在对话中展示，以节省 token 开销。文件路径为：

```
.claude/prompt_cache/full_prompt_[promptID].txt
```

系统会告知写入完成，用户可随时打开该文件审核内容、调整任意层后再继续。

### Phase 7：Prompt 精馏（系统自动）

**为什么需要精馏**：图像生成模型的交叉注意力机制在长 Prompt（>500 token）下会稀释关键约束。精馏将抽象规则转化为模型更易响应的具体视觉词汇。

**精馏过程**：

1. **Phase 1 变换**（规则 → 视觉描述符）：
   - 删除元指令和序言
   - 抽象规则转具体视觉描述符
   - 多约束规则聚类为关键词组
   - 结构规则转空间描述符

2. **Phase 2 优先级重排**：
   - 风格标识 + 布局类型 → **最高优先级**（前 15%）
   - 关键视觉约束 + 核心元素 → **高优先级**（接下来 20%）
   - 学科规范 + 次要元素 → **中优先级**（接下来 30%）
   - 质量标准（压缩）+ 细节 → **低优先级**（接下来 25%）
   - 负面约束 → **最低优先级**（最后 10%）

3. **验证 6 项关键约束是否保留**：
   - 色彩无障碍（红绿色盲安全）
   - 箭头语义（类型一致）
   - 视觉层次（3 级分明）
   - 阅读路径（流向明确）
   - 科学准确性标记（假设 = 虚线）
   - 风格标识（未被压缩破坏）

精馏 Prompt 同样**静默写入缓存文件**：

```
.claude/prompt_cache/distilled_prompt_[promptID].txt
```

系统会告知精馏后的词数与压缩率，并询问用户选择使用哪个版本。用户可打开对应文件查看内容。

**默认**：使用精馏 Prompt（250-400 词，推荐），全量 Prompt 供人工审核。两组文件同时保留，方便后续迭代对比。

## 4. 支持的学科领域

系统支持 8 大学科领域，每个领域有专属的 Prompt 模板，涵盖该领域的视觉语法、常见元素类型、色彩标准和构图规范。

### 1. 生物医学与分子生命科学
- 标准生物学视觉语法：磷脂双分子层质膜、Y 形受体、配体小分子、双膜核、线粒体嵴等
- 细胞区室明确划分：胞外、质膜、细胞质、细胞核、线粒体、内质网、高尔基体
- 信号方向语义：实线箭头（激活）、T 形终止（抑制）、虚线（间接/未知）、环形箭头（反馈）
- 翻译后修饰：磷酸化（P 小圈转移）、泛素化（Ub 标签）
- 生物学配色：暖色（活跃/致癌/促炎）、冷色（抑制/抑癌/抗炎）、绿色（结构）、灰色（基础状态）

### 2. 化学、材料与纳米科学
- 化学结构：标准键线式、球棍模型、CPK 原子着色
- 反应方案：反应物 → 中间体 → 产物，反应条件标注
- 能量/坐标图：反应坐标 vs 自由能，过渡态（‡）、活化能（ΔG‡）
- 材料架构：薄膜截面、核壳纳米颗粒、多孔框架、二维材料片层
- 合成工作流：分步表示，关键参数标注

### 3. 人工智能与计算机科学
- 神经网络层：颜色编码（蓝色=编码器，绿色=解码器，橙色=注意力，紫色=FFN，灰色=归一化）
- 数据张量：3D 等距框 + 形状标注 `[B, L, D]`
- 注意力机制：Q/K/V 并行输入，缩放点积，多头拼接
- 标准元素：Transformer Block、MoE、LoRA、位置编码、KV-Cache、RAG、扩散模型
- 流向约定：数据流=粗实线，梯度流=红虚线，残差连接=细曲线

### 4. 工程与应用物理
- 器件原理图：截面图、等距视图、俯视图
- 系统框图：功能块 + 信号/功率/流体连接
- 能源图：Sankey 图、能带图、效率级联
- 机械图：工程制图规范、受力分析、应力-应变曲线
- 电子图：IEEE/ANSI 标准符号
- 光学图：光线追踪、波前可视化

### 5. 临床医学与医疗健康
- 解剖学上下文：器官/组织半透明背景
- 疾病机制：健康→疾病进展对比（冷色→暖色过渡）
- 药物作用机制（MoA）：吸收→分布→靶点结合→信号级联→疗效
- 临床路径：决策树格式，菱形=决策点，矩形=行动，椭圆=结局
- 诊断算法：检测→结果→解读→下一步
- 配色：动脉=红，静脉=蓝，淋巴=绿，神经=黄

### 6. 环境、地球与气候科学
- 循环图：储存库（框）+ 通量（箭头，宽度∝速率）
- 生态系统：营养级垂直堆叠
- 气候系统：五大圈层交互，辐射强迫
- 地质剖面：地层柱、构造特征（断层、褶皱、不整合）
- 生物地球化学：元素储库 + 转化箭头
- 配色：大气=浅蓝，海洋=深蓝，陆地/植被=绿色，人为影响=暖色

### 7. 物理学与数学
- 费曼图：费米子=实线，光子=波浪线，胶子=螺旋线，W/Z=虚线，Higgs=点线
- 相图：一阶相变=实线，二阶=虚线，crossover=渐变
- 时空图（Penrose 图）：共形无限远边界，光锥，世界线

## 5. 七种视觉风格

每种风格都有完整的视觉指令块，包括线条质量、色彩处理、光照、纹理和整体艺术方向。

### 风格 1：Classic Vector（经典矢量——洁净期刊风）
- 扁平矢量插图，适合 Nature/Cell/Science 直接投稿
- 统一粗细轮廓（1-2pt），4-6 种纯色，零渐变
- 规则几何形状，严格模块化网格对齐
- 锐利三角形实心箭头，纯白背景 `#FFFFFF`
- 无阴影、无发光、无纹理、无 3D 效果
- 整体印象："Nature Methods 主图水准——干净、精准、普适"

### 风格 2：Hand-Drawn Sketch（手绘草图——白板思维风）
- 有机手绘感，模拟白板或实验笔记本
- 线条微抖、不完美，笔触粗细自然变化（0.5-3pt）
- 不对称手绘箭头，米白纸纹理背景 `#FAFAF5`
- 柔和自然色调：铅笔灰、墨黑、石板蓝、砖红、橄榄绿
- 标签如手写字迹，轻微不规则
- 整体印象："聪明的同事在白板上画的关键想法——非正式但极其清晰"

### 风格 3：Minimal Infographic（极简信息图——高影响力传播风）
- 杂志品质编辑级信息图，极致视觉克制
- 至少 40% 留白，仅 2-4 种颜色
- 图标化替代文字标签，≤15 个图标
- 超细线条（0.5pt），极端简约
- 8-10% 四面留白，非对称平衡
- 整体印象："3 秒读懂，Nature Communications 级品质"

### 风格 4：Scientific Illustration（科学插画——教科书写实风）
- 专业级科学插画，适合 Nature Reviews / Cell / 教科书
- 生物结构半写实渲染，准确形态学
- 柔和的自然主义渐变阴影，左上 45° 光源
- 8-12 种颜色 + 色调变化，表面纹理细节
- 三级布光（主光+补光+环境光遮蔽），无硬阴影
- 整体印象："Nature Reviews Molecular Cell Biology 的 Figure 1"

### 风格 5：Futuristic Tech（未来科技——前沿创新风）
- 深色科技美学背景 `#0A0A1A`，微妙网格纹理
- 霓虹发光线条：青色 `#00E5FF`、电蓝 `#4488FF`、品红 `#FF44AA`
- 玻璃态节点，2-4px 发光效果，数据粒子流
- 等宽字体或科技感无衬线字体
- ⚠️ 提醒：主要用于演示/海报/预印本，传统期刊投稿可能被认为不够正式
- 整体印象："NeurIPS 主舞台或 CES 发布会级"

### 风格 6：Hybrid（混合——多层视觉层次风，2025-2026 顶级趋势）
- **三层视觉体系**：
  - Layer 1（60% 视觉权重）：Clean Vector 结构框架——模块框、主流箭头、区室边界（柔和蓝灰）
  - Layer 2（25% 视觉权重）：Hand-Drawn 强调——2-3 个关键创新点（暖橙、深蓝、粗笔触）
  - Layer 3（15% 视觉权重）：Scientific Illustration 上下文——细胞/分子/组织（柔和纤细）
- 读者可三层次阅读：3 秒略读 → 30 秒学习 → 3 分钟深入研究
- 暖灰背景 `#F8F7F4`
- 整体印象："当代顶级期刊的新标准——信息层次极致分明"

### 风格 7：3D Render（3D 渲染——照片级深度风）
- 专业级 3D 科学可视化，Blender/Cinema 4D/Maya 品质

- **物理准确材质**：玻璃 IOR 1.5、金属粗糙度 0.1-0.6、生物组织次表面散射
- 透视相机（50-85mm 焦距），微妙景深
- 程序化表面纹理，微小的真实感表面瑕疵
- 整体印象："Advanced Materials 或 Cell 期刊封面级别"

## 6. 两种叙事模式

### 模式 A：机制图引擎（Mechanism Diagram）

**用途**：解释"如何运作"——因果链、过程流、结构关系、动态机制。

**叙事结构——因果链**：

```
┌──────────────────────────────────────┐
│  上下文：在哪里发生？（区室/系统边界）  │
├──────────────────────────────────────┤
│  输入 ──→ 步骤1 ──→ 中间状态          │
│              ├──→ 步骤2a ──→ ┐        │
│              └──→ 步骤2b ──→ 融合 ──→ 最终输出
├──────────────────────────────────────┤
│  关键洞察：创新点/意外发现（视觉焦点）   │
└──────────────────────────────────────┘
```

**6 个叙事节拍**：上下文 → 输入/起始状态 → 转化序列 → 关键交互 → 输出/终态 → 创新标记

### 模式 B：Graphical Abstract 引擎

**用途**：用单一自包含视觉传达论文核心贡献——面向期刊首页、社交媒体和快速理解。

**叙事结构——影响力弧**：

```
┌────────────────────────────────────────────┐
│  问题 ────→   创新    ←──── 结果             │
│  (左, 25%)  (中, 40%,   (右, 25%)           │
│   暗/小)    最大/最亮)   亮/大)              │
│                 ↓                           │
│          意义（底部 10%，宽广影响声明）        │
└────────────────────────────────────────────┘
```

**关键规则**：
- 最多 5 个视觉元素（理想 3 个）
- 恰好 1 个主色强调（创新区域）
- 至少 40% 白色/空白空间
- 无 a/b/c 面板——单一统一图像
- 必须在 200px 缩略图尺寸仍可读
- 文字 ≤ 10 词（理想 ≤ 5 词）
- 阅读路径明确：问题 → 创新 → 结果 → 意义

## 7. 六种自动布局类型

系统自动从用户输入中检测最优布局类型，无需用户手动选择。

### 类型 1：管线布局（Pipeline Layout）
```
[输入] ──→ [步骤1] ──→ [步骤2] ──→ [步骤3] ──→ [输出]
```
- **最适**：AI/ML 工作流、化学合成路线、信号转导级联、数据处理管线
- **检测关键词**：`step by step`、`pipeline`、`workflow`、`sequential`
- **宽高比**：16:9（≤5 步）或更宽（6+ 步）
- **阅读路径**：Z 形（左上→右上→左下→右下）

### 类型 2：中央枢纽布局（Central Hub Layout）
```
         [模块A]
            ↑
[模块B] ← ★ 核心 ★ → [模块C]
            ↓
         [模块D]
```
- **最适**：新框架论文、多模态融合系统、中心化处理架构
- **检测关键词**：`core`、`central`、`hub`、`our framework`、`integrates`
- **宽高比**：1:1（正方形）
- **阅读路径**：中心→上→右→下→左（顺时针）

### 类型 3：层级堆叠布局（Layered Stack Layout）
```
┌──────────────────────┐
│   输入 / 嵌入         │
├──────────────────────┤
│   编码器 / 处理       │
├──────────────────────┤
│   瓶颈 / 融合         │
├──────────────────────┤
│   解码器 / 生成       │
├──────────────────────┤
│   输出 / 预测         │
└──────────────────────┘
```
- **最适**：Transformer/CNN/RNN 架构、编码器-解码器模型
- **检测关键词**：`encoder`、`decoder`、`layer`、`stack`、`hierarchical`
- **宽高比**：9:16（纵向）
- **阅读路径**：自上而下

### 类型 4：生物空间布局（Biological Spatial Layout）
```
┌──────────────────────┐
│   胞外空间            │
├──────────────────────┤
│   质膜               │
├──────────────────────┤
│   细胞质              │
│   [信号蛋白]          │
├──────────────────────┤
│   细胞核              │
│   [转录因子] [DNA]    │
└──────────────────────┘
```
- **最适**：信号通路、膜转运、胞内运输、药物递送
- **检测关键词**：`membrane`、`cytoplasm`、`nucleus`、`compartment`
- **宽高比**：3:4 或 2:3（纵向）

### 类型 5：分支树布局（Branching Tree Layout）
```
              [根/起点]
                  │
          ┌───────┴───────┐
          ↓               ↓
     [决策A]          [决策B]
          │               │
     ┌────┴────┐     ┌────┴────┐
     ↓         ↓     ↓         ↓
  [结果]    [结果]  [结果]    [结果]
```
- **最适**：临床决策树、诊断算法、分类系统、治疗指南
- **检测关键词**：`if...then`、`decision`、`branching`、`classification`
- **宽高比**：2:3 或 3:4（纵向）

### 类型 6：循环回路布局（Cyclic Loop Layout）
```
        [过程A] ←── [过程D]
            │           ↑
            ↓           │
        [过程B] ──→ [过程C]
```
- **最适**：代谢循环（TCA、Calvin）、生物地球化学循环、反馈调控
- **检测关键词**：`cycle`、`loop`、`feedback`、`regeneration`
- **宽高比**：1:1（正方形）
- **阅读路径**：顺时针（从 12 点位置进入）

### 8.2 分辨率

所有 6 个模型均支持 **1K 分辨率**，部分模型支持更高分辨率（可通过修改 `generate_images.py` 中的参数切换）。不同模型使用不同的尺寸格式：

| 模型 | size (比例) | resolution (分辨率) |
|------|------------|-------------------|
| ERNIE-Image-Turbo | `1024x1024` / `1376x768` / `768x1376` | —（由 size 决定） |
| wan2.7-image-pro | `"1K"` / `"2K"` / `"4K"` | —（由 size 决定） |
| qwen-image-2.0 | `1024*1024` / `1280*720` / `720*1280` | —（由 size 决定） |
| z-image-turbo | `1024*1024` / `1280*720` / `720*1280` | —（由 size 决定） |
| gemini-3-pro-image | `"1:1"` / `"16:9"` / `"9:16"` 等 | `"1K"` / `"2K"` / `"4K"` |
| GPT-Image-2 | `"1:1"` / `"16:9"` / `"9:16"` 等 | `"1k"` / `"2k"` / `"4k"` |

> **⚠️ 注意：** Gemini 和 GPT-Image-2 的 `size` 填写比例（如 `"1:1"`），分辨率通过单独的 `resolution` 字段控制。Gemini 使用大写 K（`"1K"`），GPT-Image-2 使用小写 k（`"1k"`）。这与 ERNIE/wan/qwen/z-image 的格式完全不同。

### 8.3 并行生成

并行生成由 `generate_images.py`（stdlib-only Python 脚本，零外部依赖）统一编排。使用 `concurrent.futures.ThreadPoolExecutor` 实现真正的并发 I/O，各模型独立运行，单个失败不影响其他。

**关键规则**：
- 所有模型接收**相同的精馏 Prompt**
- qwen/z-image 模型设置 `prompt_extend: false` 以保留精确 Prompt 实现公平对比
- wan2.7-image-pro、gemini-3-pro-image-preview、GPT-Image-2 为异步模式——需提交任务后轮询（5 秒间隔），总超时 120 秒
- 同步模型（ERNIE/qwen/z-image）一次请求即返回结果；异步模型返回 `task_id`，通过轮询获取最终图像
- 结果以简洁 JSON 摘要形式输出，不将原始 API 响应或 Prompt 文本写入对话上下文

## 11. 跨模型对比评估

所有模型收到相同精馏 Prompt 后，系统提示从 5 个维度评估输出：

| 维度 | 评估内容 | 典型最优 |
|------|---------|---------|
| **空间准确性** | 哪个模型最好地遵循了布局指令？ | GPT-Image-2、wan2.7（thinking mode） |
| **风格保真度** | 哪个模型最好地捕捉了目标风格美学？ | qwen-image-2.0（精确 Prompt 保留） |
| **科学准确性** | 哪个模型避免了幻觉结构/连接？ | GPT-Image-2、gemini-3-pro |
| **文字质量** | 哪个模型产生了最干净的标签？ | qwen-image-2.0、GPT-Image-2 |
| **整体冲击力** | 哪个图最有效地传达了核心信息？ | 主观——全量对比 |

### 生成后处理建议

1. **选择最佳输出**：不同模型可能在不同维度表现出色
2. **文字标签**：在 Illustrator/Inkscape/PPT 中添加精确标签——AI 文字在所有模型上均不可靠
3. **科学验证**：对照已知领域事实验证所有细节
4. **色彩一致性**：匹配论文全局配色方案
5. **箭头语义**：验证所有箭头类型一致性
6. **3 秒测试**：给不熟悉该工作的同事看一眼
7. **期刊合规**：根据目标期刊调整分辨率、色彩空间和字体
8. **跨模型洞察**：跟踪哪个模型在你的学科/风格组合上表现最好

## 12. 迭代与优化

生成后系统提供以下迭代选项：

- **优化 Prompt**：调整特定措辞或细节后，重新运行全部（或选定）模型
- **调整风格强度**："更手绘" / "更少发光"
- **修复布局**：移动特定元素、添加/删除连接
- **切换风格/学科**：尝试不同方案
- **同 Prompt 重新生成**：相同 Prompt、相同模型、不同随机种子（增加 round）
- **跨模型迭代**：保留最佳模型输出，仅对表现不佳的模型调整 Prompt 后重新生成

## 13. 质量保证体系

系统内置了顶级期刊质量标准（30 条不可协商规则），作为 Prompt 的 Layer 6 写入每个生成请求。

### A. 清晰度与即时可读性
- **3 秒规则**：核心机制必须在 3 秒内被识别
- **单一流向**：统一左→右或上→下，不混合
- **元素精简**：每个元素必须证明其存在价值
- **3-7 个关键结构元素**（可逻辑分组）

### B. 三层视觉层次
- Level 1（主导）：核心通路——最大、最饱和、最粗线
- Level 2（次要）：支持细节——中等
- Level 3（三级）：上下文元素——最小、最淡
- 读者注意力序列：Level 1 (<1s) → Level 2 (2-5s) → Level 3 (10s+)

### C. 色彩标准
- 最多 6 种不同色相（不含黑白灰中性色）
- 红绿色盲安全（氘色盲 + 红色盲）：禁止纯红绿对比，确保 30% 以上亮度差，灰度下完全可辨
- 语义色彩：暖色=活跃/上调，冷色=抑制/下调，中性=结构/基线
- 色彩编码一致性：同一实体在多处出现必须颜色一致

### D. 箭头与连接语义
- 实心箭头（→）= 直接激活/促进/转化
- T 形终止（⊣）= 直接抑制/阻断
- 虚线箭头（⇢）= 间接/假设/未知机制
- 开口箭头（⟶）= 转位/运输
- 细连接线（─）= 物理结合/复合物形成
- 箭头不允许交叉（必要时使用"桥接"）

### E. 排版与标签
- 全图无衬线字体（Arial/Helvetica 等效）
- 标签 7-9pt（最终出版尺寸）
- 标签不重叠任何结构线/形状
- 所有文字水平排列（唯一例外：图表 y 轴标签）

### F. 构图与布局
- 四面至少 8% 边距
- 所有元素网格对齐
- 多面板标签位置一致

### G. 叙事结构
- 三段式故事弧：上下文 → 机制 → 结果
- 创新强调：新颖贡献必须是 Level 1 元素

### H. 科学准确性与诚信
- 空间合理性：蛋白质不能大于细胞
- 不确定性披露：假设步骤必须用虚线 + 浅色 + "?" 或"[hypothetical]"标记
- 不编造：不创造用户输入未支持的分子结构或机制细节
- 比例感：已知尺度信息必须合理尊重

### I. 技术制作标准
- 等效 300 dpi（全页 ~2100px 宽）
- RGB 色彩空间
- 缩略图可读性（400px 宽仍可辨）
- 无 AI 伪影：无乱码文字、无解剖学不可能结构、无 Escher 式空间悖论、无模糊边缘、无多光源不一致阴影

## 15. 常见问题与排查

### API 相关

| 错误 | 可能原因 | 解决方案 |
|------|---------|---------|
| `401 Unauthorized` | API Key 无效或过期 | 验证 `config.local.json` 中的 Key |
| `429 Too Many Requests` | 超出速率限制 | 等待 30 秒后重试该模型 |
| `400 Bad Request` | Prompt 过长或尺寸格式无效 | 检查尺寸格式（各模型不同）；Prompt 超过 4000 字符时截断 |
| ERNIE 尺寸无效 | 不支持的尺寸字符串 | ERNIE 仅接受 7 种固定尺寸，重新映射至最接近尺寸 |
| API MART `400` 参数错误 | `size` 传入了 `"1K"` 而非比例 | Gemini/GPT-Image-2 的 `size` 必须填比例如 `"1:1"`，分辨率用 `resolution` 字段 |
| 百炼 `PENDING` 卡住 | 异步任务未完成 | 最多轮询 120 秒；仍为 PENDING 则报告超时 |
| API MART 任务卡在 `submitted` | 异步任务未完成 | 5 秒间隔轮询，最多 120 秒；仍卡住则报告超时 |
| API MART 任务不存在 | 任务已过期 | 重新提交生成任务 |
| API MART `402` | 账户余额不足 | 前往 https://apimart.ai/keys 充值 |
| API MART `content_filter` | 安全过滤器触发 | 移除医学血腥、露骨解剖、药物滥用等内容 |
| 百炼 `task_id` 为 null | 同步模型使用了异步工作流 | wan2.7 是异步的；qwen/z-image 是同步的——使用正确模式 |
| wan2.7 轮询超时 | 模型高负载 | 提供重试；考虑使用同步替代方案（qwen/z-image） |

### 常见问题

**Q: 精馏 Prompt 丢失了关键信息怎么办？**
A: Phase 7 会将全量和精馏两种 Prompt 都写入 `.claude/prompt_cache/` 目录。如果你觉得精馏版遗漏了重要内容，可以选择使用全量 Prompt 生成。但绝大多数情况下，精馏版的效果更好（因为模型注意力更集中）。

**Q: 能否使用自己的 API Key？**
A: 可以。只需编辑 `config.local.json` 文件，填入你在百度 AI Studio、阿里云百炼或 API MART 的 API Key。未配置的平台模型会自动隐藏。

**Q: 是否可以不使用 API 而直接获取 Prompt？**
A: 可以。即使不配置任何 API Key，Phase 0-7 仍然完整运行，最终产出精馏 Prompt。你可以将精馏 Prompt 复制到任何图像生成工具（Midjourney、DALL-E、Stable Diffusion 等）使用。

**Q: 生成的图像可以直接投稿吗？**
A: 建议作为起点使用。AI 生成的文字标签不可靠，通常需要用 Illustrator/Inkscape/PPT 进行后期加工——添加精确标签、调整配色以匹配期刊要求、验证科学准确性。

**Q: 如何获得最佳效果？**
A: (1) 提供尽可能详细的描述（组件、关系、创新点）；(2) 选择 3-4 个模型进行跨模型对比；(3) 始终使用精馏 Prompt；(4) 对最佳输出进行后期编辑；(5) 记录哪个模型在你的学科/风格组合上表现最好。

**Q: 如何查看已生成的 Prompt？**
A: 全量 Prompt 和精馏 Prompt 均保存在 `.claude/prompt_cache/` 目录中，文件名为 `full_prompt_[ID].txt` 和 `distilled_prompt_[ID].txt`。你可以直接打开查看，也可以将其复制到 Midjourney、DALL-E 等其他图像生成工具中使用。

**Q: 能否跳过出图、只获取 Prompt？**
A: 可以。即使不配置任何 API Key，Phase 0-7 仍然完整运行，最终精馏 Prompt 会写入 `.claude/prompt_cache/` 目录。你只需在 Phase 8 选择"不生成"即可。

**Q: 某个模型生成失败后能否单独重试？**
A: 可以。并行生成时每个模型独立运行，单模型失败不影响其他。系统会在结果中标注失败的模型和错误原因，并提供单独重试选项，无需重新运行全部模型。

**Q: 成本如何？**
A: 6 个模型单张图像成本从 $0.01 到 $0.12 不等。选择 3-4 个模型并行生成 1K 图像，总成本通常在 $0.05-$0.30 之间。阿里百炼和百度对新用户提供数百张免费额度。
